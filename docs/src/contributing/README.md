# Contributing

Thank you for your interest in contributing to Noxu DB. This chapter documents
the development workflow, porting guidelines, testing approach, PR process, and
release procedures.

For the community code of conduct see [`code-of-conduct.md`](https://github.com/gburd/lamdb/blob/main/code-of-conduct.md)
at the project root.

## In This Chapter

1. [Build and Test](build-and-test.md) — prerequisites, cargo commands, CI requirements
2. [Porting Guidelines](porting-guidelines.md) — Java → Rust naming rules, what to preserve, what to adapt
3. [Testing Guide](testing-guide.md) — unit, integration, property-based, and fuzz testing
4. [PR Process](pr-process.md) — branch naming, commit format, review checklist
5. [Release Process](release.md) — versioning, changelog, publish steps
