# Wave 10-C — README and v2.2.1 capability-matrix refresh

**Branch**: `fix/wave10-c-readme-matrix`
**Base**: `sprint/v2.3.0-base` (built on v2.2.1, commit `a4fb2f5`).
**Date**: 2026-05-27
**Status**: complete.

## Goal

Bring `README.md` and the capability matrix in `docs/src/introduction.md`
into line with v2.2.1 reality.  Both documents predate Waves 2A–9-C and
were stale on:

- replication GA status (10 audit blockers closed; specs re-validated),
- DPL derive macros and schema evolution,
- crash-durable XA,
- sorted-dup secondaries with `associate()` and FK
  Abort/Cascade/Nullify,
- `DiskOrderedCursor`,
- the JE TCK port progress (now 243 `PORTED-EQUIVALENT` rows after
  Wave 9-C),
- the v2.2.1 test-gate count (5,625 passing),
- the actual public API used in the Quick Start example,
- the crate count (now 21 — `noxu-persist-derive` and `noxu-spec` were
  not listed in the README's 19-crate table),
- the canonical Codeberg URLs and pages-deployment URL.

Wave 10-C is documentation-only.

## Out of scope

- `Cargo.toml` (Wave 10-E may do crates.io publish prep).
- `CHANGELOG.md` (Wave 10-B is generating it from git history).
- `crates/*/tests/` (Wave 10-A is doing further JE TCK port rows).

### 2. `docs/src/introduction.md`

Capability matrix refresh:

- Renamed columns from `v1.5 | v1.6 (planned) | v2.0 (planned)` to
  `v1.5 | v1.6 | v2.0 | v2.2 (current)`.  Both v1.6 and v2.0 shipped;
  v2.2.1 is the current release.
- Reorganised rows into eight section bands so the matrix scans:
  *Storage and transactions*, *Cursors*, *Secondary databases and
  foreign keys*, *Distributed transactions (XA)*, *Collections*,
  *Serialization / DPL*, *Replication / HA*, *Test coverage*.
- Corrected one row that was speculatively marked ✅ for v2.0:
  `Cursor::get` with `Get::SearchLte` / `FirstDup` / `LastDup` is
  still `NoxuError::Unsupported` at v2.2.1 (verified against
  `crates/noxu-db/src/cursor.rs`); marked ❌ across all columns with a
  "tracked for a later sprint" pointer.
- Added rows that were missing from the old matrix:
  - `Stateright executable specs match implementation` — ⚠️ deferred
    at v2.0, ✅ at v2.2 after Wave 9-B re-validation (5 specs updated;
    no counterexamples).
  - `become_master` rejects non-`Electable` node types — ✅ at v2.2
    after Wave 9-A (was a Wave 8 RepTestBase regression).
  - Replica I/O thread auto-bootstraps on `NeedsRestore` — ✅ at v2.2
    after Wave 9-A.
  - Read-only reopen of an existing entity store
    (`allow_create=false`) — ✅ at v2.2 after Wave 7 polish.
  - Workspace test-gate counts and JE TCK PORTED-EQUIVALENT counts
    per release.
- Linked the updated rows to their wave reports
  (`internal/wave-9-a-rep-fixes.md`,
  `internal/wave-9-b-stateright-revalidation.md`,
  `internal/wave-9-c-je-tck-ports.md`) so the matrix carries its own
  citations.
- Replaced the broken Quick Start (`Environment::open(Path::new(...),
  EnvironmentConfig::default())`, `db.put(&txn, b"hello", b"world")`,
  `db.get(None, b"hello")`) with the actual v2.2.1 API, matching
  `examples/getting_started.rs`.
- Refreshed the install instruction to the same Codeberg-git pin used
  in `README.md`.
- Updated the lead paragraph from "It is being idiomatic Rust with
  zero unsafe code in library logic." to a sentence that parses, and
  to mention the narrow documented `unsafe` exceptions explicitly.
